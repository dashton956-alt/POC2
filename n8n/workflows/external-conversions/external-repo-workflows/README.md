# External Repository Workflows - n8n Import Package\n\nThis directory contains n8n workflow definitions converted from the external GitHub repository Python-based orchestrator framework. All workflows are designed to integrate with NetBox API and provide comprehensive network automation capabilities.\n\n## Workflow Categories\n\n### 1. Core Link Workflows (`/core_link/`)\n- **create_core_link.json**: Create network core links between devices\n- **modify_core_link.json**: Modify existing core link configurations\n- **terminate_core_link.json**: Safely terminate core links with cleanup\n- **validate_core_link.json**: Validate core link connectivity and performance\n\n### 2. Node Workflows (`/node/`)\n- **create_node.json**: Provision new network devices/nodes in NetBox\n\n### 3. Port Workflows (`/port/`)\n- **create_port.json**: Configure customer ports and interface assignments\n\n### 4. L2VPN Service Workflows (`/l2vpn/`)\n- **create_l2vpn.json**: Create Layer 2 VPN services between customer sites\n- **modify_l2vpn.json**: Modify L2VPN service parameters (bandwidth, MTU, QoS)\n- **terminate_l2vpn.json**: Terminate L2VPN services with safety checks\n- **validate_l2vpn.json**: Comprehensive L2VPN service validation\n\n### 5. Intent-Based Configuration (`/intent_based/`)\n- **intent_based_config.json**: Translate high-level business intents into network configurations\n\n### 6. Device Type Management (`/import_device_types/`)\n- **import_device_types.json**: Import and manage device types from vendor libraries\n\n### 7. Network Operations (`/operations/`)\n- **network_operations.json**: Centralized network operations and maintenance tasks\n\n### 8. Task Orchestration (`/tasks/`)\n- **task_orchestrator.json**: Orchestrate complex multi-step network automation tasks\n\n## Key Features\n\n### NetBox Integration\n- Full NetBox REST API integration with Token authentication\n- Comprehensive DCIM (Data Center Infrastructure Management) support\n- IPAM (IP Address Management) capabilities\n- Service and circuit management\n\n### LSO (Life-cycle Service Orchestration)\n- Automated playbook execution via LSO API\n- Configuration deployment and validation\n- Service lifecycle management\n\n### Error Handling & Validation\n- Comprehensive input validation\n- Safety checks and prerequisites validation\n- Rollback capabilities for failed operations\n- Detailed error reporting and logging\n\n### Environment Variable Support\n- `NETBOX_TOKEN`: NetBox API authentication token\n- `NETBOX_URL`: NetBox instance URL\n- `LSO_URL`: LSO automation endpoint\n- Service-specific variables for customization\n\n### Monitoring & Compliance\n- SLA monitoring and reporting\n- Performance metrics collection\n- Compliance checking and validation\n- Health monitoring and alerting\n\n## Workflow Design Patterns\n\n### Create/Modify/Terminate/Validate (CMTV) Pattern\nMost service workflows follow the CMTV pattern:\n1. **Create**: Provision new services with full validation\n2. **Modify**: Update existing services with safety checks\n3. **Terminate**: Safely decommission services with cleanup\n4. **Validate**: Comprehensive health and performance validation\n\n### Orchestrator-Core Framework\nAll workflows are designed to integrate with the orchestrator-core framework:\n- Standardized workflow IDs and tracking\n- Consistent error handling patterns\n- Common environment variable usage\n- Unified logging and monitoring\n\n## Import Instructions\n\n### Prerequisites\n1. n8n instance running with NetBox API access\n2. NetBox configured with appropriate device types and manufacturers\n3. LSO or equivalent automation platform available\n4. Environment variables configured for API access\n\n### Batch Import Process\n1. Navigate to n8n workflows section\n2. Use \"Import from File\" option for each JSON workflow\n3. Configure environment variables for your environment\n4. Test workflows in a development environment first\n5. Deploy to production with appropriate access controls\n\n### Individual Workflow Import\nEach workflow can be imported independently:\n```bash\n# Example for importing a single workflow\ncurl -X POST \"http://your-n8n-instance/api/v1/workflows/import\" \\\n  -H \"Content-Type: application/json\" \\\n  -d @create_l2vpn.json\n```\n\n## Configuration Requirements\n\n### NetBox API Configuration\n```bash\n# Required environment variables\nexport NETBOX_URL=\"https://your-netbox-instance.com\"\nexport NETBOX_TOKEN=\"your-api-token-here\"\n```\n\n### LSO Integration\n```bash\n# LSO configuration\nexport LSO_URL=\"http://your-lso-instance:8080\"\n```\n\n### Service-Specific Variables\nRefer to individual workflow documentation for service-specific environment variables.\n\n## Validation & Testing\n\n### Pre-Import Validation\n- Verify NetBox API connectivity\n- Confirm device types and manufacturers exist\n- Test environment variable access\n- Validate LSO integration\n\n### Post-Import Testing\n- Execute each workflow in test mode\n- Verify NetBox integration\n- Test error handling scenarios\n- Validate monitoring and alerting\n\n## Support & Documentation\n\n### Workflow Documentation\nEach workflow includes comprehensive inline documentation with:\n- Purpose and scope\n- Process flow diagrams\n- Environment variable requirements\n- Error handling procedures\n\n### Integration Patterns\n- RESTful API integration patterns\n- Error handling and retry logic\n- Monitoring and alerting integration\n- Security and access control patterns\n\n## Version Information\n- **Created**: 2024-12-19\n- **n8n Compatibility**: v1.0.0+\n- **NetBox Compatibility**: v3.0+\n- **Source Repository**: https://github.com/dashton956-alt/POC/tree/main/example-orchestrator/workflows\n\n---\n\n**Total Workflows**: 12\n**Total Categories**: 8\n**Framework**: orchestrator-core\n**Integration**: NetBox + LSO\n**Status**: Production Ready\n
